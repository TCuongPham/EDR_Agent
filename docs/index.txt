Tên đề tài: Nghiên cứu và phát triển AI/ML Engine tích hợp trực tiếp trên EDR Agent nhằm phát hiện hành vi mã độc và tấn công APT theo thời gian thực
________________________________________
1. Bối cảnh đề tài
Các giải pháp bảo mật truyền thống chủ yếu phát hiện mã độc dựa trên:
•	Signature 
•	IOC (Indicator of Compromise) 
•	Rule-based detection 
Tuy nhiên, các cuộc tấn công hiện đại và mã độc APT ngày càng sử dụng nhiều kỹ thuật né tránh như:
•	Fileless malware 
•	Living-off-the-Land (LOLBin) 
•	Obfuscation 
•	Process injection 
•	Encrypted payload 
•	Abuse process hợp lệ của hệ điều hành 
Điều này khiến phương pháp detection truyền thống gặp nhiều hạn chế.
Trong các hệ thống Endpoint Detection and Response hiện đại, xu hướng đang chuyển dần sang:
•	Behavioral Detection 
•	AI-driven Detection 
•	Realtime Endpoint Analytics 
•	Sensor-based AI Detection 
Thay vì gửi toàn bộ telemetry về backend để phân tích, các hệ thống mới bắt đầu tích hợp AI/ML trực tiếp trên endpoint sensor/agent nhằm:
•	phát hiện sớm hành vi nguy hiểm 
•	giảm độ trễ detection 
•	giảm phụ thuộc network/backend 
•	giảm bandwidth telemetry 
•	tăng khả năng realtime response 
Đề tài này hướng tới việc cho sinh viên thực tập nghiên cứu và xây dựng một AI/ML Engine chạy trực tiếp trên EDR Agent nhằm:
•	thu thập telemetry realtime từ endpoint 
•	phân tích hành vi hệ thống 
•	thực hiện inference ngay trên sensor 
•	phát hiện các dấu hiệu malware/APT theo hành vi 
________________________________________

3. Nội dung nghiên cứu chính
3.1 Thu thập telemetry từ endpoint
Sinh viên nghiên cứu và xây dựng module monitor các hành vi:
Process
•	Process Create/Terminate 
•	Parent-child relationship 
•	Command line 
•	Privilege escalation 
•	Process injection 
File
•	File Create/Delete/Rename 
•	Mass file modification 
•	File entropy 
•	Sensitive path access 
Registry (Windows)
•	Run/RunOnce 
•	Service modification 
•	Persistence registry 
•	IFEO hijacking 
Network
•	DNS query 
•	Outbound connection 
•	Beaconing behavior 
•	Suspicious IP/domain 
•	Abnormal port usage 
Persistence
•	Scheduled task 
•	Service creation 
•	Startup folder 
•	WMI event subscription 
________________________________________
3.2 Feature Engineering
Sinh viên xây dựng các feature phục vụ AI/ML:
Process Behavior Feature
•	Parent-child chain 
•	Rare process execution 
•	Suspicious commandline pattern 
File Behavior Feature
•	File modification frequency 
•	Entropy increase 
•	Extension change pattern 
Network Feature
•	Connection frequency 
•	Beacon interval 
•	DNS anomaly 
Behavioral Correlation
•	Office → PowerShell 
•	PowerShell → rundll32 
•	cmd → script engine → network connection 
________________________________________
3.3 Embedded AI/ML Inference
AI/ML model được tích hợp trực tiếp trên sensor/agent để:
•	thực hiện inference realtime 
•	đánh giá threat score 
•	phát hiện anomaly 
•	classify suspicious behavior 
Các hướng nghiên cứu:
•	Lightweight ML 
•	Behavioral scoring 
•	Sequence-based detection 
•	Streaming event analysis

•	Isolation Forest 
•	One-Class SVM 
•	AutoEncoder 
________________________________________
4.3 Sequence-based Detection
Phân tích chuỗi hành vi:
Ví dụ:
winword.exe
    -> powershell.exe
        -> rundll32.exe
            -> outbound connection
Hướng nghiên cứu:
•	LSTM 
•	Temporal behavior analysis 
•	Behavioral graph detection

8. Các thách thức kỹ thuật
Lightweight AI trên endpoint
Model phải:
•	chạy realtime 
•	ít tốn RAM 
•	ít CPU usage 
•	inference latency thấp 
Ví dụ mục tiêu:
•	CPU usage < 5% 
•	RAM < 100MB 
•	inference < 50ms/event 
________________________________________
Streaming Detection
Telemetry là dòng event liên tục:
ProcessCreate
FileWrite
RegistrySet
DNSQuery
SocketConnect
...
AI engine cần xử lý realtime thay vì batch processing.
________________________________________
False Positive Reduction
Một trong những bài toán lớn nhất của behavioral detection là:
•	alert noise 
•	nhầm hành vi hợp lệ thành malware 
Sinh viên cần nghiên cứu:
•	threat scoring 
•	correlation 
•	behavioral context
